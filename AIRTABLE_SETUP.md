# Airtable Database Setup Guide

Panduan untuk setup database Airtable untuk Conversion Funnel System

## Prerequisites

1. Akun Airtable (gratis atau berbayar)
2. API Key dari Airtable
3. Base baru di Airtable

## Step 1: Buat Base Baru

1. Login ke [Airtable](https://airtable.com)
2. Klik **"Create a base"**
3. Beri nama: **"Agency Nexus - Conversion Funnel"**

## Step 2: Buat Tabel Leads

### Nama Tabel: `Leads`

Buat field-field berikut:

| Field Name       | Field Type         | Options/Settings                                    |
|------------------|-------------------|-----------------------------------------------------|
| full_name        | Single line text  | Primary field                                       |
| phone_number     | Phone number      | -                                                   |
| email            | Email             | -                                                   |
| service_interest | Single line text  | -                                                   |
| status           | Single select     | Options: New, Contacted, Qualified, Converted, Lost |
| source           | Single select     | Options: landing_page, referral, social_media, direct |
| created_at       | Date              | Include time                                        |
| lead_score       | Number            | Integer, precision 0                                |
| notes            | Long text         | -                                                   |
| Appointments     | Link to table     | Link to: Appointments, Allow linking multiple       |

### Default Values:

- **status**: New
- **source**: landing_page
- **created_at**: (auto-generated by system)
- **lead_score**: (auto-calculated by system)

## Step 3: Buat Tabel Appointments

### Nama Tabel: `Appointments`

Buat field-field berikut:

| Field Name          | Field Type         | Options/Settings                                           |
|--------------------|--------------------|-----------------------------------------------------------|
| requested_service  | Single line text   | Primary field                                             |
| lead_id            | Link to table      | Link to: Leads, Allow linking to multiple records         |
| preferred_date     | Date               | Date only (no time)                                       |
| preferred_time     | Single line text   | Format: HH:mm                                             |
| appointment_status | Single select      | Options: Pending, Confirmed, Completed, Cancelled, No Show |
| created_at         | Date               | Include time                                              |
| notes              | Long text          | -                                                         |

### Default Values:

- **appointment_status**: Pending
- **created_at**: (auto-generated by system)

## Step 4: Setup Relasi Antar Tabel

Relasi sudah terbuat otomatis saat membuat field **Link to table**:

```
Leads (1) ←→ (Many) Appointments

1 Lead dapat memiliki banyak Appointments
```

## Step 5: Dapatkan API Credentials

### 5.1 Personal Access Token

1. Klik avatar Anda di kanan atas → **Account**
2. Pilih **Developer** tab
3. Klik **Generate token**
4. Beri nama: **"Agency Nexus API"**
5. Pilih scopes:
   - `data.records:read`
   - `data.records:write`
6. Pilih base yang sudah dibuat
7. Klik **Create token**
8. Copy token (akan digunakan sebagai `AIRTABLE_API_KEY`)

### 5.2 Base ID

1. Buka base Anda di Airtable
2. Klik **Help** di kanan atas
3. Pilih **API documentation**
4. Base ID terlihat di URL: `https://airtable.com/[BASE_ID]/api/docs`
5. Copy Base ID (akan digunakan sebagai `AIRTABLE_BASE_ID`)

### 5.3 Table IDs

1. Buka API documentation (seperti langkah 5.2)
2. Pilih tabel **Leads**
3. Lihat di URL atau documentation untuk Table ID
4. Ulangi untuk tabel **Appointments**

**Alternatif (lebih mudah):**
Table ID biasanya sama dengan nama tabel:
- Leads Table ID: `Leads`
- Appointments Table ID: `Appointments`

## Step 6: Set Environment Variables

Di Figma Make / Supabase, set environment variables berikut:

```bash
AIRTABLE_API_KEY=patXXXXXXXXXXXXXX
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_LEADS_TABLE_ID=Leads
AIRTABLE_BOOKINGS_TABLE_ID=Appointments
```

## Tips

1. **Field Names**: Pastikan nama field exact match dengan dokumentasi
2. **Field Types**: Gunakan tipe field yang sesuai
3. **Single Select Options**: Harus exact match dengan yang di dokumentasi
4. **Testing**: Test dengan data dummy terlebih dahulu
5. **Permissions**: Pastikan API token punya akses read & write

## Troubleshooting

### Error: "Could not find table"
- Cek Table ID sudah benar
- Pastikan API token punya akses ke base tersebut

### Error: "Invalid field name"
- Pastikan nama field di Airtable exact match
- Case sensitive (gunakan lowercase dengan underscore)

### Error: "Invalid value for field"
- Cek tipe data field sudah sesuai
- Untuk Single Select, pastikan value ada di options
