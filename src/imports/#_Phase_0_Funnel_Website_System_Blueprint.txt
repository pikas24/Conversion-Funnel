# Phase 0 — Planning & Documentation  
## Conversion Funnel System Architecture Blueprint
### Productized Funnel Framework  
**Funnel Type:** 3-Page Conversion Funnel  
**Pages:** Landing Page → Booking Page → Confirmation Page  
**Purpose:** Lead Generation & Appointment Booking  
**System Type:** Productized Funnel Architecture  

---

# 1. System Architecture Blueprint

## 1.1 System Overview

Sistem ini merupakan **conversion funnel berbasis web** yang dirancang untuk mengubah pengunjung menjadi **lead terverifikasi yang melakukan booking**.  

Arsitektur sistem dirancang menggunakan prinsip **productized funnel architecture**, sehingga:

- struktur sistem **terstandarisasi**
- dapat **direplikasi pada banyak proyek**
- **modular dan scalable**
- memisahkan **frontend, backend, dan integrasi**

Sistem terdiri dari empat layer utama:

```

Frontend Layer
↓
API / Backend Services
↓
Database Layer
↓
External Integrations

```

---

# 1.2 Funnel Page Structure

Struktur funnel terdiri dari tiga halaman utama:

```

Landing Page
↓
Booking Page
↓
Confirmation Page

```

Penjelasan fungsi halaman:

| Page | Purpose |
|-----|-----|
| Landing Page | Mengkomunikasikan value proposition dan mengarahkan user ke booking |
| Booking Page | Mengumpulkan data lead dan melakukan proses booking |
| Confirmation Page | Mengkonfirmasi booking dan memberikan langkah selanjutnya |

---

# 1.3 Frontend Pages

## Landing Page

Landing page berfungsi sebagai **entry point** funnel.

Tujuan utama halaman ini:

- menarik perhatian pengunjung
- menjelaskan value proposition
- membangun trust
- mendorong user melakukan booking

Fungsi utama:

```

Present Offer
↓
Build Trust
↓
Drive CTA Click

```

---

## Booking Page

Booking page adalah halaman **lead capture utama**.

Fungsi halaman ini:

- mengumpulkan informasi calon pelanggan
- melakukan pre-qualification
- memungkinkan user memilih jadwal

Proses utama:

```

User Input Data
↓
System Validation
↓
Booking Submission

```

---

## Confirmation Page

Confirmation page muncul setelah booking berhasil.

Fungsi utama:

- memberikan konfirmasi booking
- menampilkan detail jadwal
- memberikan langkah selanjutnya

---

# 1.4 Backend Services

Backend bertanggung jawab terhadap **business logic** funnel.

Layanan utama yang dibutuhkan:

### Booking API

Menangani proses booking user.

Fungsi:

- menerima data booking
- validasi data
- menyimpan ke database

---

### Lead Processing Service

Mengelola data lead yang masuk.

Fungsi:

- memproses data booking
- menyiapkan data untuk integrasi CRM

---

### Lead Scoring Service

Menentukan kualitas lead berdasarkan input.

Contoh faktor:

```

service_type
booking_time
user_response

```

Output:

```

lead_score

```

---

### Notification Service

Mengirim notifikasi setelah booking dibuat.

Contoh:

- email confirmation
- internal notification

---

### Webhook Service

Mengirim data lead ke sistem eksternal seperti CRM.

---

# 1.5 Database

Database digunakan untuk menyimpan informasi sistem.

Fungsi database:

```

Store booking data
Store lead data
Store lead score
Store timestamps
Store system logs

```

Database menjadi **source of truth** seluruh data funnel.

# 2. User Journey & Conversion Flow

Berikut perjalanan pengguna dalam funnel.

```

Visitor arrives at landing page
↓
User reads value proposition
↓
User views testimonials and social proof
↓
User clicks CTA button
↓
User navigates to booking page
↓
User fills booking form
↓
User selects date and time
↓
User submits booking request
↓
System validates booking data
↓
Booking stored in database
↓
Lead scoring executed
↓
System triggers notifications
↓
System sends webhook to CRM
↓
User redirected to confirmation page

```

# 6. Database Schema

## Table: bookings

| Field | Type | Description |
|------|------|------|
id | UUID | primary identifier |
booking_id | string | unique booking reference |
name | string | nama user |
phone | string | nomor telepon |
email | string | alamat email |
booking_date | date | tanggal booking |
booking_time | time | waktu booking |
service_type | string | jenis layanan |
lead_score | integer | skor lead |
status | string | status booking |
created_at | timestamp | waktu pembuatan booking |

---

### Field Explanation

**id**

Primary key untuk setiap record.

---

**booking_id**

ID unik yang digunakan sebagai referensi booking.

---

**name**

Nama lengkap user.

---

**phone**

Nomor telepon user.

---

**email**

Alamat email user.

---

**booking_date**

Tanggal yang dipilih user untuk appointment.

---

**booking_time**

Waktu booking.

---

**service_type**

Jenis layanan yang dipilih user.

---

**lead_score**

Nilai kualitas lead.

---

**status**

Status booking.

Contoh:

```

pending
confirmed
cancelled

```

---

**created_at**

Timestamp ketika booking dibuat.
